## Core Components

### 1. Unified Claudia AI CLI
A single script that consolidates:
- **Microsoft Copilot CLI**
- **Gemini CLI**
- **Perplexity CLI**
- **Grok CLI** (direct terminal power)
- **Shopify CLI** (inventory + donation data bridge)
- **Square Integration** (unified payment + tipping layer)

### 2. QR Store© — How It Works
1. A QR code opens a dynamic hosting platform.
2. Google Lens captures a picture of an item for sale.
3. The system issues a **credit** to pay for the item and presents a **tipping option**.
4. All Google Lens data is stored in the **G.L.S.©** repository for supply-chain analytics.

### 3. The "90/10" Transparency Formula
- **90%** of post-tax income is automatically reinvested into the project.
- **10%** covers a basic living stipend for the founder.
- **Full auditing:** complete transparency of all funds via on-chain or open ledger.

### 4. The "Just Above Not Happy" Algorithm
- **Equitable Priority:** human necessities (food, clothing, shelter) are prioritized by need, not wealth or tips.
- **Balance & Safety:** limits prevent hoarding or dangerous consumption.
- **Sovereign Choice:** individual rights are respected within global age-safety guidelines.

## Deployment

Project documentation is automatically published to **GitHub Pages** on every push to the `main` branch via the [`deploy.yml`](.github/workflows/deploy.yml) workflow.

| Item | Detail |
|------|--------|
| **Live docs** | `https://balancerockartist-ctrl.github.io/Blue-Star/` |
| **Trigger** | Push to `main` or manual `workflow_dispatch` |
| **Engine** | Jekyll (GitHub Pages default — renders `README.md` automatically) |
| **Permissions** | `pages: write`, `id-token: write` (OIDC deploy token) |

> **Note:** GitHub Pages must be enabled in the repository settings (Settings → Pages → Source → GitHub Actions) before the first deployment will succeed.
